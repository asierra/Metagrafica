# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this project is

MetaGráfica is a 2D descriptive vector graphics language. It compiles `.mg` source files into Encapsulated PostScript (EPS). The binary is called `mg`.

## Build commands

```bash
make          # build the mg binary and man page
make clean    # remove all build artifacts
make install  # install to /usr/local/bin and man page
```

Run an example:
```bash
./mg examples/primitives.mg
```

There is no test suite. Correctness is verified by inspecting the generated `.eps` files visually or with a PostScript viewer.

## Toolchain requirements

- `clang++` (primary) or `g++`, C++14 standard
- `flex` — required to regenerate `lexmg.cpp` from `mgpp.l`
- `pandoc` — required to regenerate the man page from `mg.1.md`

Two files are auto-generated and should not be edited directly:
- `lexmg.cpp` — generated by `flex mgpp.l`
- `version.h` — generated by `git log -1 --pretty=format:...`

## Architecture

The pipeline is: `.mg` source → **Lexer** → **Parser** → in-memory tree → **Display** → `.eps` output.

### Key abstractions

- **`GraphicsItem`** (`primitives.h`) — abstract base for every drawable element. Subclasses include `Polyline`, `Rectangle`, `Arc`, `Dot`, `Attribute`, `Transform`, `GraphicsState`.
- **`Path`** — `std::vector<point>`. The fundamental data structure; most primitives are built from paths.
- **`GraphicsItemList`** — `std::vector<std::unique_ptr<GraphicsItem>>`. Owns a sequence of graphics items.
- **`Structure`** (`structures.h`) — a named, reusable group of `GraphicsItemList`. Stored in a global `structure_map`. `StructureUser` subclasses (StructureLine, StructureRectangle, StructureArc, StructurePath) place a structure along geometric shapes.
- **`MetaGrafica`** (`structures.h`) — top-level `Structure` representing the whole document (dimensions, depth, font size).
- **`Display`** (`Display.h`) — abstract output backend. Currently only `EPSDisplay` (`EPSDisplay.h/.cpp`) is implemented.
- **`MGLexer`** (`MGLexer.h`, generated `lexmg.cpp`) — C++ class wrapping `yyFlexLexer`. Keywords and token types are initialized in `MGLexer::init_tables()`. The `.mg` keywords (PL, CR, BR, BZ, SP, MKST, OPST…) are mapped there.
- **`Parser`** (`Parser.h/.cpp`) — reads a `.mg` file using `MGLexer`, builds the `MetaGrafica` tree. Entry point is `Parser::parse()`.
- **`Matrix`** (`matrix.h/.cpp`) — 3×3 homogeneous coordinate matrices for 2D transforms. `PredefinedMatrix` enum names the four matrix slots (MTLC local, MTST structure, MTPP plume position, MTPT path, MTRS repeat structure).

### Adding a new primitive

1. Add a token in `mgpp.l` (`keyword_map` in `init_tables()`).
2. Add a `GI_*` enum value in `primitives.h` and a new `GraphicsItem` subclass.
3. Handle the token in `Parser::parsePrimitive()` or one of the helper `parse*` methods in `Parser.cpp`.
4. Implement `draw(Display &)` to call the appropriate `Display` virtual methods.
5. Implement the corresponding virtual method in `EPSDisplay` to emit PostScript.

## Code style

Follows [Orthodox C++](https://gist.github.com/bkaradzic/2e39896bc7d8c34e042b): no RTTI (`-fno-rtti`), no exceptions (`-fno-exceptions`). Uses `std::unique_ptr` for ownership, raw pointers for non-owning references. Compiled with `-Wall -Wpedantic -O3`.
