Formato vectorial 2D. El elemento básico no es el pixel sino el punto geométrico x y.

El siguiente elemento es la ruta (path) formada por una lista finita de puntos. 

Los primitivos gráficos usan paths para dibujar líneas, polígonos, etc.

Las rutas pueden ser manipuladas: invertidas, normalizadas para encajar en un rectángulo, concatenadas, etc.

Una manera de hacer las rutas poderosas es el uso de distintos tipos de splines soportados: cúbicos, Catmull-Rom, Bezier y Splines cuadráticos. No todos son soportados nativamente por los lenguajes de salida, para lo cual se usa algún recurso, como convertirlos a uno que sí este soportado, o desarrollarlos en rutas más grandes con puntos interpolados de acuerdo al spline.

Otro concepto importante es que MG es una máquina de estados gráficos.

Un concepto tal vez obsoleto es que la posición de la "pluma" como en los antiguos plotters, se mantiene como parte del estado. En los nombres de las clases se nota un lenguaje de computer graphics de los años 1980's.

El espacio geométrico se normaliza de 0 a 1.

Se mantiene un aspect ratio definido por el usuario y que sea fácil para el usuario tener objetos dentro de estructuras que no sean deformados. En el código ya hay varios trucos que se utilizan para eso pero tal vez hay mejores estrategias y más modernas que deberíamos explorar.

Parte del estado gráfico son los atributos como el color, el ancho de línea, el patrón de línea o relleno.

Las transformaciones lineales se mantienen en el estado gráfico con matrices 2D en el espacio de coordenadas homogéneas y hay matrices para varias aplicaciones, como la posición y el tamaño y orientación de las estructuras. En la versión actual se manejan prefijos como LC (local), ST (estructuras), PP (posición actual de pluma), PT (rutas).

Las estructuras permiten asociar primitivas, atributos y matrices, lo que permite crear objetos derivados (no primitivos) más complejos. De hecho cada archivo MG se alberga en una estructura.

La combinación de estructuras con matrices son la estrategia más poderosa de MG para crear gráficos complejos.

El texto es un tipo especial de elemento gráfico pues como los lenguajes nativos manejan fuentes e hileras de caractéres, no es necesario dibujarlas. Se puede manejar su posición igual que se manejan las rutas pero algunos de sus atributos muy específicos de texto se manejan aparte, aunque es deseable que también se puedan manipular por transformaciones lineales dentro de las estructuras.

Los generadores fue una idea reciente en la versión actual, para generar secuencias de números controlando sus posiciones, rutas y líneas paralelas (ticks). Algunas de estas ideas fueron trucos para crear objetos como ejes y gráficas, pero la idea nueva es que exista una estructura axis con propidades que encapsule más sencillo ese resultado sin necesidad de tantas líneas.

Ideas ya contempladas en la especificación son el uso de variables y parámetros en las estructuras.

Se propone usar loops para difersos efectos, pero antes hacer un inventario de los loops que ya existen implícitamente, para refinar las ideas de una nueva sintáxis para simplificar y hacer más poderosa esta idea.

También hay la idea de usar recursión controlada para algunos efectos gráficos interesantes.

Una vez que tengamos la especificación final y su gramática, evaluar la posibilidad de implementar un nuevo parser, tal vez con herramientas modernas para ello, así como ya usamos flex para el léxico. El objetivo son un lenguaje fácil de entender, práctico, lógico y poderoso para usuarios humanos y también, por qué no, que un asistente de IA ayude a crear gráficas profesionales para publicaciones científicas y técnicas.
